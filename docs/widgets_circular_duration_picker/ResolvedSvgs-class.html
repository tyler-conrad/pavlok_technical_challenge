ResolvedSvgs

ResolvedSvgs class Null safety

This class holds the 'bedtime' and 'wakeUp' SVG images that are asynchronously loaded in the CircularDurationPicker class.

Using a Future within the m.CustomPainter and drawing in Future.then callback caused a bug where the m.Canvas was disposed before the callback was executed and the canvas that was referenced in the closure. The callback was invalidated and therefore threw an exception.

To fix this issue I use a nullable ResolvedSvgs variable that only takes on a value once the Future for both SVGs has been resolved.
